# i18n / bilingual UI

ProctorPal supports English (`en`) and Korean (`ko`) at the UI layer.

## Where strings live

All translatable strings are in **`src/lib/i18n.tsx`**, in two parallel
dictionaries: `en` and `ko`. Every key must exist in both. Using a key that's
missing in one locale falls back to the English string at runtime, but the
TypeScript type for `t()` only allows keys that exist in the `en` dictionary,
so you'll get a compile error if you add to `ko` without adding to `en`.

## Using a string

In a client component:

```tsx
"use client";
import { useT } from "@/lib/i18n";

export function MyButton() {
  const t = useT();
  return <button>{t("common.submit")}</button>;
}
```

With variables (use `{name}` placeholders):

```tsx
t("runner.questionOf", { n: 3, total: 10 });
// EN: "Question 3/10"
// KO: "문제 3/10"
```

In a server component, import `tFor`:

```tsx
import { tFor } from "@/lib/i18n";
const heading = tFor("ko", "auth.signInTitle"); // for SSR-rendered KO content
```

## Adding a new string

1. Add a key to **both** `en` and `ko` blocks in `src/lib/i18n.tsx`.
2. Use `t("your.key")` in your component.
3. The key naming convention is `feature.thing` (e.g., `home.subtitle`,
   `runner.startTest`, `paused.tab_blur`).

## Switching language

The `<LanguageToggle />` component is rendered in the root layout (top-right
corner of every page). User selection persists in `localStorage` under
`proctorpal_locale`.

Default is English. To set a different default for a fresh user (e.g., based
on browser `Accept-Language`), update `DEFAULT` in `src/lib/i18n.tsx` or
hydrate the locale in a server-side cookie.

## What's translated so far

Fully translated:
- Home page
- Login / Signup
- Student entry, waiting room, runner header, start gate, paused overlay,
  navigation, done page
- Teacher header nav

Not yet translated (as of last edit):
- Test editor (settings, sections, questions, publish)
- Live monitor dashboard
- Grading dashboard
- Class detail (assignments, manual entry form, CSV import)
- Gradebook

To translate any of these, follow the "Adding a new string" steps. The full
list of question-type names, status badges, and proctoring rule labels are
prime candidates if you teach in Korean.

## Notes on Korean strings

Education vocabulary used (Korean - meaning):
- 시험 - test/exam
- 학생 - student
- 학번 - student ID number
- 선생님 / 교사 - teacher
- 수업 - class (instructional)
- 반 - class (group of students)
- 채점 - grading/scoring
- 제출 - submit
- 문제 - question/problem (test item)
- 답 - answer
- 일시정지 / 재개 - pause / resume
- 미리보기 - preview
- 접속 코드 - access code

If a Korean term feels off in classroom context, it's a one-line dictionary
edit - no code changes needed.
